PSD → 预制体 操作指南

Cocos Creator 2.4.15 · cocos-psd-prefab-2x 扩展

一句话流程：装好依赖 → 打开面板 → 选 PSD → 点转换。工具会按图层命名规范 自动导出图片、建节点树、生成预制体，并用 MD5 去重让相同图片只导一次。

0. 准备：安装依赖（只需一次）

把 cocos-psd-prefab-2x 放进工程 packages/ 后,在该扩展目录执行一次安装 (若用的是含依赖的 Release 整包则可跳过):

cd <你的工程>/packages/cocos-psd-prefab-2x
npm install
没装依赖就点转换，面板会提示 failed to load converter deps … run npm install …，照提示装一下即可。

2. 打开扩展面板

A

在 Cocos Creator 顶部菜单选 PSD 工具 / PSD 转预制体

弹出可停靠面板「PSD → 预制体」。

3. 选 PSD，点转换

1

点「选择 PSD」

在弹出的文件框里选你的 .psd，路径会回填到上面的输入框。

2

（可选）填输出目录 / 预制体名

输出目录默认 PSD（即写到 assets/PSD/）。预制体名留空则用 PSD 文件名。

3

（可选）勾选 MD5 去重

默认开。开启后内容相同的图片只导入一张，多个节点 / 多个预制体共用，跨多次运行也生效。

4

（可选）勾选「裁剪图片四周透明边」

默认勾选。有些图层是一小块图外加一大圈透明，直接导出会得到超大图、尺寸和位置都不对。勾上后会自动裁掉全透明边，用真实内容当尺寸与位置——尺寸、位置都更准，结果显示「裁剪 N」。若不想改变原图就取消勾选;也可对单个图层用 @nocrop 保留留白，九宫格 sp_ 永不裁剪。

5

（可选）勾选「压平只含1个子节点的空容器」

默认勾选。PSD 的"组/拷贝"常只包一个子节点，层级又深又乱。勾上后会把这种无组件、只有 1 个子的空容器合并掉，并保留外层有意义的名字，节点树更扁平、好写代码，结果显示「压平 N」。
另外工具会自动让同父节点下节点名唯一(重名加 _2)，并在 8 字符截断时保留末尾数字(形状590→XingZ590)，确保 getChildByName 能定位——这是写代码的硬保证，无开关。

6

（可选）勾选「合并重叠的装饰图」实验

默认不勾。一个元素常由"外发光 + 内层"两张几乎一样的图叠成，勾上后会把同位置、尺寸相近的相邻兄弟图只保留一张。会改变外观，按需开启，结果显示「合并 N」。

7

（可选）勾选「检测缓存图片是否还在」

默认不勾。去重表会记住已导出过的图；如果你之后手动删了某张 PNG，默认会复用缓存、不再重导，导致预制体引用到丢失的图。勾上后，复用每张图前会先检查文件是否还在，被删过就自动重新导出，结果会显示「重导 N」。平时不勾更快；删过图 / 换过素材 / 发现有图显示不出来时勾一次自愈。

8

点「转换为预制体」

完成后面板显示 节点 / 新图 / 复用 / 裁剪 / 压平 / 合并 / 重导 / 忽略 数量与预制体路径，资源库自动刷新。

4. 产物在哪

assets/<输出目录>/
├── prefabs/<预制体名>.prefab        ← 双击打开就是还原的界面
├── textures/<图层名>.png(.meta)      ← 每张图带 spriteFrame
└── .psd_assets.json                  ← MD5 去重表（自动维护，勿手删）

5. 关键点是怎么保证的

① 位置精确对齐

每张图按图层真实像素包围盒导出(自动裁透明边)，锚点统一 (0.5,0.5)，节点坐标 = 包围盒中心换算到 Cocos 坐标系。导出盒与算坐标盒是同一个，做到像素级对齐（实测误差<0.5px）。

② 无用节点不创建

注释 / 参考 / 临时前缀、隐藏层、全透明、空包围盒、空组在解析阶段直接过滤，既不建节点也不导图。

③ 相同图片按 MD5 复用

对裁切后的 PNG 字节算 MD5，命中持久化去重表就复用已有 spriteFrame 的 uuid，不再写新图。

④ 代码友好命名

节点名全英文 + 类型前缀(Img_/Label_/Node_) + 后缀≤8 + 同父唯一，getChildByName 稳定可用。

7. 常见问题

现象处理
点转换提示 failed to load converter deps在扩展目录 cocos-psd-prefab-2x 跑 npm install
预制体没出现在资源面板稍等资源库刷新，或手动右键 assets/PSD → Reimport
预制体有图显示不出来（删过图后）缓存仍指向被删的图。勾选「检测缓存图片是否还在」再转一次，会自动重导缺失的图（结果显示「重导 N」）
某些图层没生成检查是不是命中忽略规则（//、ref_、tmp_、隐藏、全透明、空层）
某个图片节点尺寸特别大 / 图标偏到角落该图层是“小图 + 一大圈透明”，包围盒没裁紧。保持「裁剪图片四周透明边」勾选并重转即可（结果会显示「裁剪 N」）
位置有偏差确认图层无多余图层样式（投影/外发光会撑大包围盒）；文字层若要像素级一致，建议直接做成图
同图没被复用确认两图像素完全一致；带 @unique 的会强制独立；缩放过的图 MD5 不同
节点名没语义（形状/矩形/图层）这是 PS 默认层名,工具只能保证唯一+英文。让美术给代码要用的层起有意义的名字(见命名规范第十二节)
